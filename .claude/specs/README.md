# specs（仕様書フォルダ）

## このフォルダに書くこと

プロジェクトの **仕様書**（何を作るか・どう動くべきか）を置く。
要件・スコープ・アーキテクチャ・API・データモデルなど、「実装の設計図」になるドキュメントが対象。

## 書くべきもの

- 機能仕様 / MVPスコープ
- アーキテクチャ・コンポーネント構成
- API設計・データモデル
- 確定した設計判断（なぜその方式を選んだか）

## 書かないもの

- 検討中の議論・賛否・レビュー結果 → `discussions/`（このフォルダ内のサブフォルダ）へ
- コードそのもの → リポジトリ本体へ

## ファイル命名規則

```
YYYY-MM-DD-vN-{内容}.md
```

例: `2026-05-27-v1-ui-variant-preview-agent-mvp.md`

- `YYYY-MM-DD`: 作成日
- `vN`: バージョン（仕様を改訂したら v2, v3 と増やす）
- `{内容}`: 仕様の対象を表す短い名前

## 運用ルール

- 仕様を改訂したら**新しいバージョンのファイルを作る**（古い版は残す）。
- ファイルを追加・更新したら必ず [summary.md](summary.md) を更新する。
