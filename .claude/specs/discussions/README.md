# discussions（ディスカッションフォルダ）

`specs/` の配下に置く、**特定の仕様に対する** 検討・議論・レビューの記録フォルダ。

## このフォルダに書くこと

仕様を確定させる前後の **検討・議論・レビューの記録** を置く。
「なぜこうするか」を考えた過程、賛否、リスク、未解決の論点など、思考のログが対象。

## 書くべきもの

- 批判的レビューの結果（仕様の穴・リスク・矛盾の指摘）
- 設計判断の検討メモ（選択肢の比較、トレードオフ）
- 未解決の論点・宿題（あとで決めること）
- 議論の経緯（いつ・何を・なぜ決めたか）

## 書かないもの

- 確定した仕様そのもの → 親フォルダ `../`（specs直下）へ
- コード → リポジトリ本体へ

## specs（親フォルダ）との違い

- `../`（specs直下） = **結論**（何を作るか）
- `discussions/`（このフォルダ） = **過程**（なぜそう決めたか・何が論点か）

議論で結論が出たら、その結論を親フォルダ `../` の仕様書に反映する。

## ファイル命名規則

```
YYYY-MM-DD-{内容}.md
```

例: `2026-05-27-mvp仕様-批判的レビュー.md`

## 運用ルール

- ファイルを追加・更新したら必ず [summary.md](summary.md) を更新する。
- どの仕様（specs の版）に対する議論かを各ファイル冒頭に明記する。
