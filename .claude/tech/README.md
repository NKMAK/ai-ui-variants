# tech（技術ドキュメントフォルダ）

## このフォルダに書くこと

このプロジェクトで **使用する技術の選定理由と使い方** を置く。原則 **1技術 = 1ファイル**。
各ファイルは「何を・なぜ採用し・このプロジェクトでどう使い・何に注意するか」を記録する。

仕様（何を作るか）は `../specs/` に、技術（何でどう作るか）はここに、と役割を分ける。

## 書くべきもの

- 採用技術ごとの役割・採用理由・具体的な使い方・注意点
- フォルダ構成とその意図（[folder-structure.md](folder-structure.md)）

## 書かないもの

- 何を作るか / どう動くべきか（機能仕様・スコープ・API設計） → `../specs/` へ
- 検討中の議論・賛否・レビュー → `../specs/discussions/` へ
- コードそのもの → リポジトリ本体へ

## ファイル命名規則

- 1技術 = 1ファイル。kebab-case の `.md`（例: `vite.md` / `preact-signals.md`）
- フォルダ構成の説明は `folder-structure.md`
- ファイル索引は [summary.md](summary.md)

## 運用ルール

- 技術を追加・変更したら、該当ファイルを更新し**必ず [summary.md](summary.md) を更新する**。
- 採用判断の背景が変わったら該当ファイルに追記する（古い判断も経緯として残してよい）。
